# روبوٹ آپریٹنگ سسٹم 2

روبوٹ آپریٹنگ سسٹم (ROS) سافٹ ویئر لائبریریوں اور ٹولز کا ایک سیٹ ہے جو روبوٹ ایپلی کیشنز بنانے میں مدد کرتا ہے۔ یہ روایتی معنوں میں کوئی آپریٹنگ سسٹم نہیں ہے، بلکہ روبوٹ سافٹ ویئر لکھنے کے لیے ایک لچکدار فریم ورک ہے۔ ROS 2 ROS کی دوسری نسل ہے، اور اسے اپنے پیشرو کے مقابلے میں زیادہ مضبوط، محفوظ اور قابل توسیع بنانے کے لیے ڈیزائن کیا گیا ہے۔

## ROS 2 کیا ہے؟

ROS 2 ROS 1 کا مکمل دوبارہ لکھا گیا ورژن ہے، اور اسے اصل فریم ورک کی کچھ حدود کو دور کرنے کے لیے ڈیزائن کیا گیا ہے۔ ROS 2 کے کچھ اہم مقاصد یہ ہیں:

*   **متعدد روبوٹ کی حمایت:** ROS 2 کو متعدد روبوٹ والے سسٹم بنانے اور ان کا انتظام کرنا آسان بنانے کے لیے ڈیزائن کیا گیا ہے۔
*   **ریئل ٹائم سپورٹ:** ROS 2 ریئل ٹائم ایپلی کیشنز کے لیے بہتر سپورٹ فراہم کرتا ہے، جو بہت سے روبوٹکس کاموں کے لیے اہم ہے۔
*   **بہتر سیکیورٹی:** ROS 2 میں متعدد سیکیورٹی خصوصیات شامل ہیں جو ROS 1 میں دستیاب نہیں ہیں۔
*   **کراس پلیٹ فارم سپورٹ:** ROS 2 کو کراس پلیٹ فارم بنانے کے لیے ڈیزائن کیا گیا ہے، اور اسے Linux، macOS اور Windows پر استعمال کیا جا سکتا ہے۔

## ROS 2 کے بنیادی تصورات

ROS 2 متعدد بنیادی تصورات پر مبنی ہے جنہیں روبوٹ ایپلی کیشنز بنانا شروع کرنے سے پہلے سمجھنا ضروری ہے۔

*   **نوڈز:** ایک نوڈ ایک ایسا عمل ہے جو کچھ حساب کرتا ہے۔ ROS 2 میں، ہر پروگرام ایک نوڈ ہوتا ہے۔
*   **موضوعات:** ایک موضوع ایک نامی بس ہے جس پر نوڈز پیغامات کا تبادلہ کر سکتے ہیں۔ موضوعات غیر متوازی، ایک سے بہت سے مواصلات کے لیے استعمال ہوتے ہیں۔
*   **خدمات:** ایک خدمت متوازی، ایک سے ایک مواصلات کے لیے ایک درخواست-جواب میکانزم ہے۔
*   **اعمال:** ایک عمل ایک طویل عرصے سے چلنے والا کام ہے جو فیڈ بیک فراہم کرتا ہے اور اسے روکا جا سکتا ہے۔

## ROS 2 مواصلاتی نظام

ROS 2 اپنے مواصلاتی نظام کے لیے ڈیٹا ڈسٹری بیوشن سروس (DDS) معیار کا استعمال کرتا ہے۔ DDS ایک مڈل ویئر معیار ہے جو ریئل ٹائم، قابل توسیع اور محفوظ مواصلات کے لیے ڈیزائن کیا گیا ہے۔ DDS کا استعمال کرتے ہوئے، ROS 2 متعدد خصوصیات فراہم کرنے کے قابل ہے جو ROS 1 میں دستیاب نہیں ہیں، جیسے سروس کے معیار (QoS) کی ترتیبات اور نوڈز کی خودکار دریافت۔

## ROS 2 ٹولز

ROS 2 متعدد کمانڈ لائن ٹولز کے ساتھ آتا ہے جو روبوٹ ایپلی کیشنز کو تیار کرنے اور ڈیبگ کرنے کے لیے ضروری ہیں۔ کچھ عام ٹولز میں شامل ہیں:

*   `ros2 run`: یہ ٹول ایک پیکیج سے ایک نوڈ کو چلانے کے لیے استعمال ہوتا ہے۔
*   `ros2 topic`: یہ ٹول موضوعات کا معائنہ کرنے اور ان کے ساتھ تعامل کرنے کے لیے استعمال ہوتا ہے۔
*   `ros2 node`: یہ ٹول نوڈز کا معائنہ کرنے اور ان کے ساتھ تعامل کرنے کے لیے استعمال ہوتا ہے۔
*   `rqt`: یہ ٹول ایک گرافیکل یوزر انٹرفیس (GUI) ہے جسے ROS 2 ایپلی کیشنز کو دیکھنے اور ڈیبگ کرنے کے لیے استعمال کیا جا سکتا ہے۔

## ایک ROS 2 ورک اسپیس بنانا

ROS 2 ایپلی کیشنز بنانا شروع کرنے سے پہلے، آپ کو ایک ورک اسپیس بنانے کی ضرورت ہے۔ ایک ورک اسپیس ایک ڈائریکٹری ہے جس میں آپ کے ROS 2 پیکیجز ہوتے ہیں۔ ایک ورک اسپیس بنانے کے لیے، آپ درج ذیل کمانڈز استعمال کر سکتے ہیں:

```bash
mkdir -p ros2_ws/src
cd ros2_ws
colcon build
```

## ایک سادہ پبلشر اور سبسکرائبر لکھنا

ROS 2 کے بنیادی تصورات کو واضح کرنے کے لیے، آئیے ایک سادہ پبلشر اور سبسکرائبر بنائیں۔ پبلشر ایک موضوع پر ایک پیغام شائع کرے گا، اور سبسکرائبر پیغام وصول کرے گا اور اسے کنسول پر پرنٹ کرے گا۔

### Python پبلشر

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalPublisher(Node):
    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello, ROS 2! %d' % self.i
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    minimal_publisher = MinimalPublisher()
    rclpy.spin(minimal_publisher)
    minimal_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### C++ پبلشر

```cpp
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>

int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);
  auto node = rclcpp::Node::make_shared("minimal_publisher");
  auto publisher = node->create_publisher<std_msgs::msg::String>("topic", 10);
  std_msgs::msg::String message;
  message.data = "Hello, ROS 2!";
  publisher->publish(message);
  rclcpp::spin(node);
  rclcpp::shutdown();
  return 0;
}
```

پبلشر کو چلانے کے لیے، آپ درج ذیل کمانڈ استعمال کر سکتے ہیں:

```bash
ros2 run <package_name> minimal_publisher
```

## ROS 1 بمقابلہ ROS 2

ROS 2 ROS 1 کے مقابلے میں ایک اہم بہتری ہے، اور یہ نئی روبوٹ ایپلی کیشنز کے لیے تجویز کردہ ورژن ہے۔ ROS 1 اور ROS 2 کے درمیان کچھ اہم اختلافات یہ ہیں:

*   **مواصلات:** ROS 2 اپنے مواصلاتی نظام کے لیے DDS کا استعمال کرتا ہے، جبکہ ROS 1 ایک کسٹم TCP پر مبنی پروٹوکول استعمال کرتا ہے۔
*   **بلڈ سسٹم:** ROS 2 اپنے بلڈ سسٹم کے طور پر `colcon` کا استعمال کرتا ہے، جبکہ ROS 1 `catkin` کا استعمال کرتا ہے۔
*   **Python API:** ROS 2 میں Python API ROS 1 میں API کے مقابلے میں زیادہ Pythonic ہے۔
*   **لانچ سسٹم:** ROS 2 میں لانچ سسٹم ROS 1 میں لانچ سسٹم کے مقابلے میں زیادہ لچکدار اور طاقتور ہے۔

ROS 2 کے کلیدی تصورات اور ٹولز کو سمجھ کر، آپ اپنی روبوٹ ایپلی کیشنز بنانے کے لیے اچھی طرح سے تیار ہو جائیں گے۔
